// Technical Reference
Developer Documentation

NexIntent uses Model Context Protocol to interpret compliance intent natively. Review our conceptual endpoint architecture.

Core Philosophy: Zero Data Retention

Standard compliance structures force software providers to cache internal business transactions. NexIntent completely circumvents this hazard. The system generates ephemeral runtime specifications — classifying intent, mapping schema, and executing read-only queries — without persisting any client financial data at any stage of the pipeline. Every operation is stateless by design.

Intent Route API — Core Endpoint

All NexIntent queries pass through a single authenticated endpoint. The system handles intent classification, schema mapping, and result formatting internally.

// POST /api/v1/intent/route
Authorization: Bearer NEX_SECURE_TOKEN_HEX
Content-Type: application/json

// Request Body
{
  "intent_prompt": "Audit high-risk accounts matching SYSC 6.1.1...",
  "mask_pii": true,
  "fca_schema_version": "v4.2"
}
Security Architecture — TLS & Key Management

All API endpoints enforce TLS 1.3. TLS 1.2 and below are explicitly blocked at the load balancer. Cryptographic keys are managed via Azure Key Vault with HSM backing. Client API keys are stored with AES-256 encryption and are revocable instantly from the dashboard.
